MODELO BASE¬∂

Un modelo base es un modelo simple que se utiliza como punto de referencia para evaluar el rendimiento de modelos m√°s avanzados. Sirve como una referencia b√°sica para determinar si los modelos complejos ofrecen mejoras significativas en el rendimiento. La elecci√≥n de un modelo base proporciona un est√°ndar m√≠nimo que otros modelos deben superar para justificar su complejidad adicional.

En concreto, en este trabajo se ha optado para la construcci√≥n de un modelo base que impute a la clase mayoritaria, es decir, que predice la clase m√°s frecuente en un conjunto de datos desbalanceado. Adem√°s, se establece una semilla para la generaci√≥n de n√∫meros aleatorios de 12345 para garantizar que el comportamiento sea reproducible y si se ejecuta el c√≥digo varias veces con esta semilla se obtengan los mismos resultados.

Se√±alar que al no tratarse de un algoritmo de regresi√≥n (DummyClassifier) se emplean los datasets no estandarizados.

En primer lugar se define la funci√≥n 'evaluate_model', que toma tres par√°metros y muestra las siguientes m√©tricas:

  • Puntuaci√≥n ROC-AUC: Esto se imprime solo si se proporciona ypred_proba. Eval√∫a el √°rea bajo la curva de Caracter√≠stica Operativa del Receptor, lo cual es √∫til para problemas de clasificaci√≥n binaria.

  • Precisi√≥n (Accuracy): Calcula e imprime la precisi√≥n del modelo utilizando las etiquetas reales (ytest) y las etiquetas predichas (ypred).

  • Informe de clasificaci√≥n: Imprime el informe de clasificaci√≥n, que incluye precisi√≥n, exhaustividad, puntuaci√≥n F1 y soporte para cada clase.

√ÅRBOL DE DECISI√ìN¬∂

Un √°rbol de decisi√≥n es un modelo predictivo en aprendizaje autom√°tico supervisado y no param√©trico que utiliza una estructura de √°rbol para representar reglas de decisi√≥n (empleado tanto clasificaci√≥n y regresi√≥n). Cada nodo interno del √°rbol representa una prueba sobre una caracter√≠stica del conjunto de datos, las ramas representan los resultados de la prueba, y las hojas indican las predicciones.

El objetivo es crear un modelo que prediga el valor de una variable objetivo al aprender reglas de decisi√≥n simples inferidas a partir de las caracter√≠sticas de los datos. Estos √°rboles se construyen dividiendo el conjunto de datos de manera recursiva con el objetivo de maximizar la homogeneidad en t√©rminos de la variable objetivo.

Para m√°s informaci√≥n se puede consultar la informaci√≥n que proporciona la web de la librer√≠a scikit learn acerca de los √°rboles de decisi√≥n: Informaci√≥n adicional

Otra diferencia es el manejo de las caracter√≠sticas categ√≥ricas: XGBoost trata por defecto las variables categ√≥ricas como num√©ricas ordenadas, requiriendo codificaci√≥n one-hot para evitar problemas de interpretaci√≥n mientras que LightGBM maneja caracter√≠sticas categ√≥ricas f√°cilmente, permitiendo la especificaci√≥n de columnas categ√≥ricas y dividiendo por igualdad.
